# MySQL Database Schema Documentation

This document describes the relational database structure for the **AI Smart Campus Assistant**. The backend uses SQLAlchemy ORM to automatically verify and create these tables.

---

## 1. Entity Relationship Summary

The schema contains 6 primary tables:
*   `users` connects to `complaints` (One-to-Many), `chat_history` (One-to-Many), and `notifications` (One-to-Many).
*   `complaints` connects to `complaint_status_history` (One-to-Many) and `notifications` (One-to-Many).
*   `users` also links to `complaint_status_history` as the administrator auditor who modified status.

---

## 2. Table Schemas

### 2.1. `users` Table
Stores authentication profiles for both students and administration personnel.

| Column | Data Type | Nullable | Key | Default | Description |
| :--- | :--- | :---: | :---: | :--- | :--- |
| `id` | INT | No | PK (AI) | | Unique user primary identifier. |
| `username` | VARCHAR(50) | No | Unique | | Login handle (Indexed). |
| `email` | VARCHAR(100) | No | Unique | | Email address for status updates. |
| `password_hash` | VARCHAR(255) | No | | | Pbkdf2/SHA256 salted password hash. |
| `role` | VARCHAR(20) | No | | `'student'` | Access level: `'student'` or `'admin'`. |
| `full_name` | VARCHAR(100) | Yes | | | Display name of the user. |
| `department` | VARCHAR(50) | Yes | | | College department (e.g. Computer Science). |
| `created_at` | DATETIME | No | | `UTC_TIMESTAMP` | Account registration timestamp. |

---

### 2.2. `campus_info` Table
Stores college administrative information (knowledge base) loaded by the chatbot RAG search.

| Column | Data Type | Nullable | Key | Default | Description |
| :--- | :--- | :---: | :---: | :--- | :--- |
| `id` | INT | No | PK (AI) | | Unique record identifier. |
| `category` | VARCHAR(50) | No | Index | | Category: `'courses'`, `'fees'`, `'exams'`, etc. |
| `question_keywords` | VARCHAR(255) | No | | | Comma-separated keyword queries to match. |
| `answer_content` | TEXT | No | | | Detailed response served by the bot. |

---

### 2.3. `complaints` Table
Stores campus maintenance complaints reported by students.

| Column | Data Type | Nullable | Key | Default | Description |
| :--- | :--- | :---: | :---: | :--- | :--- |
| `id` | VARCHAR(36) | No | PK | | Unique complaint ID, formatted as `CMP-XXXXXX`. |
| `student_id` | INT | No | FK | | Reference to user who reported it (`users.id`). |
| `title` | VARCHAR(150) | No | | | Subject title of the complaint. |
| `description` | TEXT | No | | | Complete details describing the issue. |
| `location` | VARCHAR(100) | No | | | Classroom, floor, hostel block. |
| `image_path` | VARCHAR(255) | Yes | | | Cloudinary secure URL or relative path on server. |
| `predicted_category`| VARCHAR(50) | Yes | | | AI classified category (e.g. Furniture Damage). |
| `confidence_score` | FLOAT | Yes | | | AI prediction accuracy rating (0.0 to 1.0). |
| `status` | VARCHAR(20) | No | | `'Pending'` | Status: `'Pending'`, `'In Progress'`, `'Resolved'`. |
| `created_at` | DATETIME | No | | `UTC_TIMESTAMP` | Submission timestamp. |
| `updated_at` | DATETIME | No | | `UTC_TIMESTAMP` | Timestamp of last modification. |

---

### 2.4. `complaint_status_history` Table
Tracks status audits and administrator remarks logs over time.

| Column | Data Type | Nullable | Key | Default | Description |
| :--- | :--- | :---: | :---: | :--- | :--- |
| `id` | INT | No | PK (AI) | | Unique timeline record identifier. |
| `complaint_id` | VARCHAR(36) | No | FK | | Reference to target complaint (`complaints.id`). |
| `status` | VARCHAR(20) | No | | | Target updated status. |
| `remarks` | TEXT | Yes | | | Explanatory note or instructions from admin. |
| `updated_by` | INT | Yes | FK | | ID of admin who updated status (`users.id`). |
| `updated_at` | DATETIME | No | | `UTC_TIMESTAMP` | Log record timestamp. |

---

### 2.5. `chat_history` Table
Maintains chatbot question-and-answer archives.

| Column | Data Type | Nullable | Key | Default | Description |
| :--- | :--- | :---: | :---: | :--- | :--- |
| `id` | INT | No | PK (AI) | | Unique message record identifier. |
| `student_id` | INT | No | FK | | Reference to chatting student (`users.id`). |
| `user_message` | TEXT | No | | | Input message sent by the student. |
| `bot_response` | TEXT | No | | | Output text response generated by AI. |
| `timestamp` | DATETIME | No | | `UTC_TIMESTAMP` | Conversational exchange timestamp. |

---

### 2.6. `notifications` Table
Stores notifications generated for users (such as issue status transitions).

| Column | Data Type | Nullable | Key | Default | Description |
| :--- | :--- | :---: | :---: | :--- | :--- |
| `id` | INT | No | PK (AI) | | Unique notification identifier. |
| `user_id` | INT | No | FK | | Target recipient user ID (`users.id`). |
| `complaint_id` | VARCHAR(36) | Yes | FK | | Optional connected complaint ID (`complaints.id`). |
| `message` | TEXT | No | | | Display message content. |
| `is_read` | BOOLEAN | No | | `False` | Unread state check. |
| `created_at` | DATETIME | No | | `UTC_TIMESTAMP` | Notification creation timestamp. |
